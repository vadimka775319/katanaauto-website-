# Автозагрузка на сервер

## Вариант 1: Авто-загрузка при сохранении (FTP)

Запустите и оставьте открытым — при каждом сохранении файла он будет загружаться:

```powershell
.\deploy-auto.ps1
```

или двойной клик на **deploy-auto.bat**

**Нужен FTP** — если на VPS нет FTP, этот вариант не сработает.

## Вариант 2: GitHub Actions (авто при push)

При каждом push в репозиторий файлы автоматически загрузятся на сервер.

### Настройка

1. Создайте репозиторий на GitHub
2. Загрузите туда файлы сайта
3. В репозитории: Settings → Secrets and variables → Actions
4. Добавьте секреты:
   - `SERVER_HOST` = `95.81.124.28`
   - `SERVER_USER` = `root`
   - `SERVER_PASSWORD` = ваш пароль
   - `SERVER_PORT` = `22` (или порт из панели)

5. При каждом `git push` сайт будет автоматически выкладываться

## Вариант 3: Планировщик задач Windows

Автозагрузка по расписанию (например, раз в день):

1. Win + R → `taskschd.msc` → Enter
2. Создать задачу → Триггер: ежедневно
3. Действие: Запуск программы
4. Программа: `powershell.exe`
5. Аргументы: `-File "C:\Users\user\katanaauto-website\deploy.ps1"`
